## Этап 1. SQL

  

Вести **sql-academy.org** и **Stepik** параллельно, оба — от начала до конца, не выборочно.

  

- [ ] Пройти [sql-academy.org](https://sql-academy.org) целиком (теория + практика по каждой теме)

- [ ] Пройти [stepik.org/course/63054](https://stepik.org/course/63054/promo) — «Интерактивный тренажер по SQL», 22 урока по порядку

  

### Темы, на которых точно нужно задержаться (были явные пробелы в диагностике)

- [ ] Транзакции: что это, зачем `ROLLBACK`, `COMMIT`

- [ ] Isolation levels: dirty read, phantom read (хотя бы на уровне смысла)

- [ ] Индексы: как работают, композитный индекс и порядок столбцов

- [ ] `EXPLAIN` / `EXPLAIN ANALYZE` — как читать план запроса

- [ ] N+1 проблема — связать с ORM (Eloquent/SQLAlchemy), не как абстрактный термин

- [ ] `WHERE` vs `HAVING`

- [ ] `GROUP BY` vs `DISTINCT`

- [ ] `UNION` vs `UNION ALL`

- [ ] Денормализация — когда оправдана

- [ ] Репликация и шардинг — на уровне "зачем нужно"

- [ ] Хранимые процедуры — плюсы/минусы vs логика в коде приложения

  

**Проверка готовности:** переписать без ошибок запрос из диагностики (JOIN двух таблиц + агрегация + фильтр по сумме + сортировка) и объяснить его EXPLAIN.

## Этап 3. Backend-архитектура (строго после Этапа 2 — нужны классы)

  

- [ ] Пройти [Бесплатный курс Backend от MediaTen](https://mediaten.ru/education/backend) целиком: SOLID, паттерны проектирования, DDD, IoC/DI, MVC, чистая архитектура, жизненный цикл запроса, устройство фреймворков

  

### Темы, которые стоит явно сверить после курса (были пустые ответы в диагностике)

- [ ] REST API + HTTP-глаголы (verbs) — что делает каждый

- [ ] Идемпотентность HTTP-методов — какие идемпотентны, какие нет

- [ ] Статус-коды: 401 vs 403, 404 vs 410, 500 vs 503

- [ ] ORM — плюсы/минусы vs чистый SQL

- [ ] middleware — своя аналогия, проговорить вслух несколько раз (тема регулярно забывается)

- [ ] REST vs RPC vs GraphQL — на уровне отличий

- [ ] Message queue (RabbitMQ/Redis Queue) — на примере: отправка email после регистрации

- [ ] Rate limiting — зачем нужен на API

- [ ] Горизонтальное vs вертикальное масштабирование

- [ ] Load balancer — зачем, если есть один сервер

  

### Зафиксировать (важная путаница из диагностики)

- [ ] **Аутентификация** = проверка личности (логин+пароль). **Авторизация** = проверка прав доступа. Было перепутано местами — повторить до автоматизма.

## Параллельный трек A. Python-хендбук

  

Идти по [«Основы Python»](https://education.yandex.ru/handbook/python) от начала до конца по оглавлению. Можно вести параллельно с любым из этапов выше.

  

### Главы, где будут закрываться конкретные пробелы — не проскакивать бегло

- [ ] Гл. 3.3 «Списочные выражения. Модель памяти для типов» — здесь настоящий list comprehension (в диагностике был перепутан с декоратором) и mutable/immutable типы

- [ ] Гл. 4.3 «Рекурсия. Декораторы. Генераторы» — настоящие decorators и generators/`yield`

- [ ] Гл. 5 — ООП: объектная модель, магические методы, наследование, исключения

  

### Темы вне хендбука — добрать отдельно после его прохождения (хендбук не идёт в конкурентность)

- [ ] `async`/`await` — глубже базового понимания

- [ ] GIL (Global Interpreter Lock) — хотя бы "почему потоки не ускоряют CPU-bound код"

- [ ] Потоки vs процессы в Python

- [ ] Context manager (`with`) на уровне протокола `__enter__`/`__exit__` — в диагностике был неверный синтаксис

- [ ] `is` vs `==` — проверить отдельно, даже если частично зацепится в гл. 3.3

- [ ] Сериализация/десериализация (Pydantic, DRF Serializer) — связать с уже понятой темой ORM
